The AIUML Notation

A universal language for software architecture that bridges human understanding and AI processing

Type System

Abbreviated types for efficiency:

  • s = string
  • i = int
  • f = float
  • b = bool
  • L[T] = List
  • D[K,V] = Dict
  • O[T] = Optional

Modifiers

Rich semantics with single characters:

  • + public
  • - private
  • # protected
  • % static
  • ~ async
  • & abstract
  • @ property

Relationships

Express complex architectures:

  • < Inheritance
  • <<i>> Interface
  • <<a>> Abstract
  • -> Association
  • o-- Aggregation
  • *-- Composition

Complete Example

Readable Format

aiuml::class v0.1.0b

// Service with inheritance and interface implementation
UserService<BaseService<<i>>Cacheable,<<i>>Auditable{
  -repository:UserRepository
  -cache:CacheService

  +UserService(repo:UserRepository,cache:CacheService)
  +authenticate(username:s,password:s):O[User]
  ~+fetchUserAsync(id:i):User  // Async method
  %+validateEmail(email:s):b    // Static method
  &+processRequest():*         // Abstract method
}

// Interface definition
<<i>>Cacheable{
  +cacheKey():s
  +cacheTtl():i
}

// Standalone function for procedural code
#hashPassword(password:s,salt:s):s

High Compression Format (Same Content)

aiuml::class v0.1.0b
UserService<BaseService<<i>>Cacheable,<<i>>Auditable{-repository:UserRepository-cache:CacheService+UserService(repo:UserRepository,cache:CacheService)+authenticate(username:s,password:s):O[User]~+fetchUserAsync(id:i):User%+validateEmail(email:s):b&+processRequest():*}<<i>>Cacheable{+cacheKey():s+cacheTtl():i}#hashPassword(password:s,salt:s):s

High compression removes whitespace and comments while preserving all architectural information. Context-efficient format that's easier for AI assistants to parse due to its consistent, delimiter-based structure.

Get Started

Quick Start

# Clone and setup
git clone https://github.com/aiuml/developer.git
cd aiuml
pip install -e .

# Initialize project
aiuml init
aiuml status

# Build model and explore
aiuml model
aiuml nav @*

MCP Integration

# Add to Claude Desktop config
{
  "mcpServers": {
    "aiuml": {
      "command": "python",
      "args": ["/path/to/start_mcp.py"]
    }
  }
}

Our Approach

AIUML Developer is an experiment in improving how AI understands code. We believe that providing structured architectural context will help LLMs work more effectively with codebases.

What We're Testing

  • Can pre-computed architecture improve AI code understanding?
  • Does a proposed standardized notation help LLMs reason about code?
  • Can SQL queries provide better code insights?
  • How does progressive caching affect performance?

What We've Learned

  • Structured context often helps LLMs navigate complex codebases
  • SQL queries enable powerful pattern discovery
  • Progressive understanding reduces overhead
  • Project boundaries matter for context management

This is experimental software exploring new ideas. Results vary based on codebase complexity and specific use cases. We're learning and improving with community feedback.
